| Nhóm chức năng               | Frontend                                                 | Backend 1                                         | Backend 2                                     |
| ---------------------------- | -------------------------------------------------------- | ------------------------------------------------- | --------------------------------------------- |
| Tiền xử lý                   | Thiết kế màn hình upload và hiển thị thống kê/biểu đồ    | Phụ trách thuật toán và API                       | Hỗ trợ kiểm tra nội dung báo cáo nếu cần      |
| Tập phổ biến và luật kết hợp | Màn hình nhập ngưỡng, hiển thị itemsets/rules            | Phụ trách thuật toán Apriori và association rules | Hỗ trợ review kết quả                         |
| Gom cụm                      | Màn hình K-Means, Kohonen, hiển thị cụm/centroid/neuron  | Phụ trách K-Means và Kohonen                      | Hỗ trợ review kết quả                         |
| Tập thô                      | Màn hình hiển thị kết quả nếu được tích hợp vào frontend | Hỗ trợ review API                                 | Phụ trách Rough Set, reduct, rules            |
| Phân lớp                     | Màn hình Bayes, Laplace, Gain, Gini, cây quyết định      | Hỗ trợ review API                                 | Phụ trách Decision Tree, Naive Bayes, Laplace |

## Ghi chú đánh giá đóng góp

- Tỷ lệ 30% cho Frontend phù hợp vì thành viên này phụ trách toàn bộ giao diện, luồng upload file, gọi API và hiển thị kết quả cho tất cả thuật toán.
- Tỷ lệ 35% cho Backend 1 phù hợp vì phụ trách nhiều nhóm thuật toán xử lý dữ liệu: tiền xử lý, tập phổ biến, luật kết hợp và gom cụm.
- Tỷ lệ 35% cho Backend 2 phù hợp vì phụ trách tập thô, phân lớp và thêm phần báo cáo Word, trong đó báo cáo cần tổng hợp kết quả của cả nhóm.
